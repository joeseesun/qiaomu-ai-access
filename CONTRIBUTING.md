# Contributing

Thanks for improving `qiaomu-ai-access`.

## Good Contributions

- Better signal-report wording.
- Safer consent and privacy-hygiene boundaries.
- Tests for trigger routing or report rendering.
- Documentation for official, compliant AI access paths.

## Not Accepted

- VPN/proxy evasion tutorials.
- IP, account-region, payment, KYC, CAPTCHA, or fingerprint bypass steps.
- Code that changes a user's system, browser, network, certificates, DNS, hosts, or account settings automatically.

## Development

```bash
npm install
npm run check
```

Open a pull request with:

- summary
- verification commands
- any safety-boundary changes
